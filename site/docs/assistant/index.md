# AI-ассистент

AI-ассистент — ключевой функциональный модуль проекта. Он позволяет пользователю
задать вопрос на естественном языке и получить точный ответ, основанный исключительно
на материалах базы знаний.

## Зачем нужен ассистент

Традиционный поиск по сайту работает по ключевым словам: если пользователь не знает
точного термина, он не найдёт нужную страницу. AI-ассистент понимает смысл вопроса
и находит релевантный фрагмент даже при нестандартной формулировке. Это сокращает
время поиска и снижает вероятность ошибки из-за пропущенного документа.

## Почему RAG, а не обычный чат-бот

Обычный языковой чат-бот генерирует ответы из обученных весов и может придумывать
факты (галлюцинации). В данном проекте используется архитектура **RAG**
(Retrieval-Augmented Generation):

1. Модель **сначала ищет** нужные фрагменты в векторном индексе базы знаний.
2. Только после этого **генерирует ответ**, опираясь исключительно на найденный текст.
3. К ответу прилагается **список источников** — конкретные страницы базы, откуда взяты данные.

Это делает ответы проверяемыми и надёжными — важное требование для материалов по ИБ.

## Как это работает технически

```
Вопрос пользователя
       ↓
Преобразование вопроса в эмбеддинг (векторное представление)
       ↓
Поиск ближайших фрагментов в векторном индексе базы знаний
       ↓
Передача найденных фрагментов + вопроса в языковую модель
       ↓
Ответ + ссылки на страницы-источники
```

## Ожидаемый формат ответа

- Краткий и точный ответ на заданный вопрос.
- Ссылки на страницы базы знаний, откуда взята информация.
- Явное сообщение "данных недостаточно", если нужный материал отсутствует в базе.

## Связь с модулем автодополнения (C#)

Пока пользователь вводит запрос, серверный модуль на C# предлагает автодополнение
по уже известным терминам и заголовкам статей. Это ускоряет ввод и помогает
пользователю правильно сформулировать запрос ещё до его отправки ассистенту.

## Текущее состояние

Архитектура спроектирована, интерфейсная часть подготовлена. Векторный индекс
и подключение языковой модели будут реализованы после наполнения основной базы знаний
предметным контентом по ИБ.
