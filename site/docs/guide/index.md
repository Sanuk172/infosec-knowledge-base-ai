# Разделы базы знаний

Этот раздел объясняет, как организована база знаний и по какому принципу
в неё добавляются материалы по информационной безопасности.

## Зачем нужна такая структура

Единая, предсказуемая структура решает три задачи:

- **Для читателя** — понятно, где искать нужную информацию: теория отделена от практики.
- **Для AI-ассистента** — однотипное оформление упрощает индексацию и повышает точность ответов.
- **Для команды** — новые авторы сразу понимают, куда добавлять материал и в каком формате.

## Назначение разделов

### Раздел 1 — Теория и стандарты
Содержит вводные и концептуальные материалы: определения терминов, описание стандартов
(ISO 27001, NIST, ГОСТ), модели угроз и основы ИБ. Целевая аудитория — новые
сотрудники и студенты, которые знакомятся с предметной областью.

### Раздел 2 — Практика и инструкции
Содержит прикладные материалы: пошаговые инструкции, чеклисты, разбор типовых ошибок
и готовые рабочие процедуры. Целевая аудитория — специалисты, которым нужен быстрый
оперативный справочник.

## Принцип наполнения

1. Одна страница — одна чёткая тема. Это ключевое требование для корректной работы AI-поиска.
2. Каждая статья начинается с цели и целевой аудитории.
3. Структура заголовков единообразна — ассистент опирается на неё при извлечении фактов.
4. Внутренние ссылки между статьями повышают полноту ответов ассистента.

## Жизненный цикл статьи

```
Автор пишет Markdown → git push → GitHub Actions собирает сайт →
страница появляется на портале → индексатор добавляет её в RAG-базу →
ассистент использует материал в ответах
```
