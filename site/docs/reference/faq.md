# FAQ — Часто задаваемые вопросы

Ответы на вопросы об устройстве портала, логике его работы и принципах
взаимодействия компонентов системы.

## Зачем использовать сайт, а не обычный Word-документ или папку с файлами?

Статический сайт на VitePress даёт три преимущества перед папкой с документами:
- **Версионирование** — каждое изменение зафиксировано в Git с автором и датой.
- **Автопубликация** — правка в файле → push → сайт обновлён без ручного деплоя.
- **Индексируемость** — структурированный HTML проще парсить для RAG-поиска, чем DOCX.

## Почему контент хранится в Markdown, а не в базе данных?

Markdown-файлы в Git — это подход "контент как код" (Docs as Code). Он позволяет
применять к документации те же процессы, что и к программному коду: код-ревью,
ветки, откат изменений. Кроме того, Markdown без труда конвертируется в любой формат
и легко читается AI-моделями при индексации.

## Как публиковать новые статьи?

1. Создайте `.md`-файл в папке `guide/` (теория) или `guide/section-2` (практика).
2. Добавьте ссылку на страницу в файл конфигурации навигации `.vitepress/config.mts`.
3. Выполните `git push` — GitHub Actions автоматически соберёт и опубликует сайт.

## Когда ассистент начнёт отвечать по новым статьям?

После того, как новая страница опубликована, индексатор обрабатывает её текст,
создаёт векторное представление и добавляет его в RAG-базу. Только после этого
ассистент сможет использовать материал в ответах.

## Для чего нужен модуль C# в составе проекта?

Модуль реализует **автодополнение запросов** — быстрый поиск по префиксу в
отсортированном списке терминов и заголовков. Бинарный поиск работает за
`O(log n)` и обеспечивает мгновенные подсказки даже на большом словаре.
Это часть серверной логики, отвечающая за UX поискового интерфейса.

## Можно ли изменить оформление, не трогая контент?

Да. Стили вынесены в `theme/custom.css`, конфигурация навигации — в `config.mts`,
а тексты статей хранятся в `.md`-файлах. Все три слоя независимы.
